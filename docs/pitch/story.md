# Pitch Story

## Problem

Security teams often have two disconnected surfaces:

- runtime monitoring for active threats
- repository scanning for code and dependency risk

CyberLens presents them in one workspace.

## Narrative

1. The operator opens Live Monitor and sees current service health, incidents, and suspicious traffic.
2. An attack pattern or regression is detected in the runtime stream.
3. The operator pivots to the supply-chain view to inspect repository risk and AI-generated remediation guidance.
4. The same system tracks live analysis progress, code findings, and repository summary output.

## Why It Matters

- One tool covers runtime and source risk.
- Realtime streams reduce waiting and blind spots.
- AI summaries make triage faster without hiding raw evidence.

## Differentiators

- Unified SOC + supply-chain workflow
- GCP-aware monitoring surface
- ADK trace visibility for long-running code scans
- Support for both GitHub repos and local projects
